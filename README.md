## 🔧 What I've Built

### [DisputeFlow](https://dispute-flow.app) — Event-Driven Dispute Processing
> `Spring Boot` `Kafka` `FastAPI` `PostgreSQL` `React` `Docker` `DigitalOcean`

Full-stack microservices system automating dispute document processing across multiple bank portals. Multi-bank per-thread-pool architecture, reason code mapping (AMEX S01, PMTC 98), rules-based decision engine, Kafka async event streaming, Docker + Nginx + SSL on DigitalOcean.

**→ [dispute-flow.app](https://dispute-flow.app)**

---

### [TagAssist](https://tagassist.dev) — Hybrid ML Fraud Validation Engine
> `Python` `XGBoost` `MiniLM` `Flask` `Docker` `Gunicorn` `Nginx`

Combines XGBoost (structured fraud signals) + MiniLM sentence embeddings to auto-approve high-confidence fraud tag change requests. Trained on 43 real annotated cases. 3-class output: TRUEF / VCAC / NOT. Safely auto-approves **8–15% of cases** during fraud spikes. Production-hardened with CPU-only PyTorch, worker timeout tuning, and optimized Docker image.

**→ [tagassist.dev](https://tagassist.dev)**

---

### [Similar Invoice Generator](https://github.com/Neetansh1457) — Internal Ops Tool
> `Spring Boot` `Java` `React` `PDF Generation`

Replaced manual Word-template invoice creation (15–20/day per investigator, 3–5 min each). Generates Amazon-style PDFs from structured inputs in under 1 minute. Projected at team scale: **~1,000 investigator-hours/week saved**, **$400K+/month** in recoverable chargeback value.

---

### [Hold Queue](https://github.com/Neetansh1457/Hold-Queue) — Systems Design Case Study
> `Systems Design` `Operational Engineering` `Product Thinking`

Identified Snooze mechanism misuse causing silent case aging, ownership loss, and avoidable chargeback SLA failures at Amazon. Designed a Shift-Bound Hold Queue with strict guardrails. Documented as a full product spec with failure mode analysis — shared with team leadership.

**→ [Read the case study](https://github.com/Neetansh1457/Hold-Queue)**
